# 开发文档

## 1. 项目概述

本项目是一个基于 React + TypeScript + Node.js + MySQL 的企业级数据库管理系统，提供了全面的数据库管理功能、用户管理、权限控制、性能优化、统计分析等企业级功能。系统设计遵循模块化、可扩展的原则，支持多种数据库操作场景，满足企业级应用的需求。

### 1.1 技术栈

- **前端**: React 18.3.1 + TypeScript + Ant Design 6.3.5 + React Router 6 + Axios
- **后端**: Node.js 16+ + Express 4.18+ + TypeScript + MySQL 8.0+
- **数据库**: MySQL 8.0+ (支持主从复制)
- **缓存**: Redis 7.0+ (可选，用于API缓存和会话管理)
- **异步处理**: Worker Threads (用于耗时任务处理)
- **认证**: JWT (JSON Web Token) + MFA (Multi-Factor Authentication)
- **部署**: Docker (可选) + Nginx

### 1.2 系统架构

系统采用前后端分离架构：

- **前端**: 单页应用 (SPA)，使用 React 构建，通过 API 与后端通信
- **后端**: RESTful API 服务，处理业务逻辑和数据库操作
- **数据库**: 主从架构，支持读写分离
- **缓存**: Redis 缓存热点数据，提高响应速度

## 3. 核心功能模块

### 3.1 SQL查询增强

**功能**: 执行中断、EXPLAIN分析、批量执行、语法校验

**API接口**:

- `POST /api/sql/execute` - 执行SQL查询
    - 参数: `{ sql: string, limit?: number, timeout?: number }`
    - 返回: `{ success: boolean, data?: any[], fields?: any[], error?: string, executionTime?: number }`

- `POST /api/sql/format` - 格式化SQL
    - 参数: `{ sql: string }`
    - 返回: `{ formatted: string }`

- `POST /api/sql/validate` - 语法校验
    - 参数: `{ sql: string }`
    - 返回: `{ valid: boolean, error?: string }`

- `POST /api/sql/explain` - 执行计划分析
    - 参数: `{ sql: string }`
    - 返回: `{ plan: any[] }`

- `POST /api/sql/batch` - 批量执行SQL
    - 参数: `{ sqls: string[] }`
    - 返回: `{ results: any[], summary: { total: number, success: number, failed: number, executionTime: number } }`

**实现细节**:

- 使用 Worker 线程处理耗时的 SQL 执行
- 实现查询超时机制，防止长时间占用数据库连接
- 支持查询结果的分页和导出

**相关文件**:

- `src/pages/SqlQuery.tsx` - SQL查询页面
- `server/index.ts` - SQL执行相关API
- `server/worker.js` - SQL执行Worker

## 5. 部署与运行

### 5.1 环境要求

- **操作系统**: Windows Server 2016+, Ubuntu 18.04+, CentOS 7+
- **Node.js**: 16.0+
- **MySQL**: 8.0+
- **Redis**: 7.0+ (可选)
- **内存**: 4GB+ (推荐)
- **CPU**: 2核+ (推荐)
- **磁盘**: 50GB+ 可用空间

### 5.2 安装步骤

1. **克隆项目**

    ```bash
    git clone <repository-url>
    cd <project-directory>
    ```

2. **安装依赖**

    ```bash
    npm install
    ```

3. **配置环境变量**
   创建 `.env` 文件：

    ```env
    PORT=3001
    DB_HOST=localhost
    DB_PORT=3306
    DB_USER=root
    DB_PASSWORD=your_password
    DB_NAME=your_database
    JWT_SECRET=your_jwt_secret
    ```

4. **启动开发服务器**

    ```bash
    npm run dev
    ```

5. **构建生产版本**
    ```bash
    npm run build
    ```

### 5.3 生产部署

1. **构建前端**

    ```bash
    npm run build
    ```

2. **启动后端服务**

    ```bash
    npm run start
    ```

3. **配置反向代理**
   使用 Nginx 或 Apache 配置反向代理，将前端静态文件和后端API请求转发到相应的服务。

## 6. 开发规范

### 6.1 代码风格

- 使用 TypeScript 严格模式
- 遵循 ESLint 规则
- 使用 Prettier 格式化代码
- 组件命名使用 PascalCase
- 函数命名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

### 6.2 提交规范

- 提交信息使用语义化提交格式
- 分支命名使用 feature/xxx 或 fix/xxx
- 代码审查后合并到主分支

### 6.3 测试规范

- 单元测试覆盖核心功能
- 集成测试验证API接口
- 端到端测试验证完整流程

## 7. 安全注意事项

- 使用 HTTPS 加密传输
- 密码使用 bcrypt 加密存储
- API接口使用 JWT 认证
- 输入验证防止 SQL 注入和 XSS 攻击
- 权限控制防止越权访问
- 定期备份数据库
- 监控异常登录和操作

## 8. 性能优化建议

- 使用 Redis 缓存热点数据
- 数据库索引优化
- 分页查询减少数据传输
- 前端使用虚拟滚动处理大数据
- 后端使用 Worker 线程处理耗时任务
- API 响应时间监控和优化
- 定期清理日志和临时数据

## 9. 故障排除

### 9.1 常见问题

1. **数据库连接失败**
    - 检查 MySQL 服务是否启动
    - 验证数据库配置是否正确
    - 检查网络连接

2. **API 响应缓慢**
    - 检查数据库查询性能
    - 查看服务器资源使用情况
    - 检查是否有死锁

3. **前端页面加载缓慢**
    - 检查网络带宽
    - 优化前端代码
    - 使用浏览器开发者工具分析性能

4. **权限验证失败**
    - 检查 JWT 令牌是否过期
    - 验证用户权限配置
    - 检查权限资源配置

### 9.2 日志查看

- 操作日志：`GET /api/logs/operations`
- 错误日志：`GET /api/logs/errors`
- 服务器日志：查看服务器控制台输出
