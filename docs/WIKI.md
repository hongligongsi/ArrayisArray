# DB Admin 项目百科

## 项目概述

DB Admin 是一个功能强大、界面友好的数据库管理系统，提供全方位的数据库操作和监控能力。该项目采用现代化的技术栈，支持多种数据库操作和系统监控功能，为开发人员和数据库管理员提供高效的工作平台。

## 历史发展

## 技术架构

### 前端技术栈

- **React 18**: 现代化的前端框架，提供组件化开发能力
- **TypeScript**: 类型安全的 JavaScript 超集
- **Ant Design**: 企业级 UI 组件库
- **Recharts**: 数据可视化图表库
- **Vite**: 快速的前端构建工具
- **React Router**: 客户端路由管理
- **Axios**: HTTP 客户端，用于 API 请求

### 后端技术栈

- **Node.js**: JavaScript 运行时环境
- **Express**: Web 应用框架
- **MySQL**: 关系型数据库
- **bcrypt**: 密码加密库
- **JWT**: 认证令牌生成和验证
- **cors**: 跨域资源共享中间件
- **mysql2**: MySQL 客户端

## 核心功能详解

### 数据管理模块

#### 表管理

- **表创建**: 支持通过界面创建新表，设置字段类型、约束等
- **表结构查看**: 可视化展示表的结构信息
- **索引管理**: 创建、删除索引，优化查询性能
- **表操作**: 支持清空表、删除表等操作

#### 数据浏览

- **分页展示**: 高效的分页机制，支持大数据量浏览
- **数据编辑**: 直接在界面上编辑数据记录
- **数据删除**: 支持单条或批量删除数据
- **搜索过滤**: 支持按字段搜索和过滤数据

#### SQL查询

- **语法高亮**: SQL 语句语法高亮显示
- **查询执行**: 执行任意 SQL 语句
- **结果展示**: 表格形式展示查询结果
- **批量执行**: 支持批量执行多条 SQL 语句

### 用户认证模块

#### 用户登录

- **身份验证**: 用户名密码验证
- **JWT 认证**: 基于令牌的认证机制
- **会话管理**: 安全的会话管理

#### 密码管理

- **修改密码**: 用户可以修改自己的密码
- **忘记密码**: 通过邮箱重置密码
- **密码加密**: 使用 bcrypt 进行密码加密存储

#### 安全特性

- **密码策略**: 密码强度验证
- **令牌安全**: 重置令牌哈希存储
- **使用限制**: 令牌使用次数和状态控制

### 系统监控模块

#### 操作日志

- **日志记录**: 记录所有用户操作
- **日志查询**: 支持按时间、用户、操作类型查询
- **日志归档**: 定期归档历史日志

#### 错误日志

- **错误捕获**: 捕获系统运行时错误
- **错误分析**: 提供错误堆栈和上下文信息
- **告警机制**: 关键错误自动告警

#### 性能监控

- **查询性能**: 监控 SQL 查询执行时间
- **资源使用**: 监控系统资源使用情况
- **连接状态**: 监控数据库连接状态

### 数据分析模块

#### 数据可视化

- **图表展示**: 支持折线图、柱状图等多种图表
- **趋势分析**: 展示数据变化趋势
- **实时更新**: 数据实时更新和展示

#### 报表功能

- **报表生成**: 生成各类业务报表
- **导出功能**: 支持导出 PDF、Excel 等格式
- **自定义报表**: 支持自定义报表模板

## 部署与配置

### 环境要求

- **Node.js**: >= 16.0.0
- **MySQL**: >= 5.7
- **npm**: >= 8.0.0
- **操作系统**: Windows, Linux, macOS

### 部署步骤

1. **克隆代码仓库**

    ```bash
    git clone https://github.com/hongligongsi/ArrayisArray.git
    cd ArrayisArray
    ```

2. **安装依赖**

    ```bash
    npm install
    ```

3. **启动服务**

    ```bash
    # PowerShell 环境
    .\start.ps1

    # 命令提示符环境
    start.bat

    # 或手动启动
    npm start
    ```

4. **访问系统**
    - 前端: http://localhost:3002
    - 后端 API: http://localhost:3001

### 数据库配置

项目支持 MySQL 数据库，配置文件位于 `server/.db-connections.json`。

```json
{
    "host": "localhost",
    "port": 3306,
    "user": "root",
    "password": "your_password",
    "database": "db_admin"
}
```

## 安全机制

### 认证与授权

- **JWT 认证**: 使用 JSON Web Token 进行身份验证
- **基于角色的访问控制**: 细粒度的权限管理
- **会话管理**: 安全的会话创建和验证机制

### 数据安全

- **密码加密**: 使用 bcrypt 算法进行密码加密
- **输入验证**: 严格的输入验证，防止 SQL 注入
- **CSRF 保护**: 跨站请求伪造防护

### 令牌安全

- **令牌哈希**: 重置令牌使用 bcrypt 哈希存储
- **使用状态**: 令牌使用状态标记，防止重复使用
- **过期机制**: 令牌自动过期，提高安全性

## 常见问题

### 启动问题

**问题**: 端口 3002 被占用
**解决方案**: Vite 会自动切换到可用端口（如 3003），请查看控制台输出的实际端口

**问题**: 数据库连接失败
**解决方案**: 检查数据库配置文件，确保 MySQL 服务正在运行

### 权限问题

**问题**: 无法执行某些操作
**解决方案**: 检查用户权限设置，确保拥有足够的权限

**问题**: 忘记密码无法重置
**解决方案**: 联系管理员重置密码，或检查邮箱配置

### 性能问题

**问题**: 页面加载缓慢
**解决方案**: 检查网络连接，清除浏览器缓存

**问题**: 查询执行时间长
**解决方案**: 优化 SQL 查询，添加适当的索引
